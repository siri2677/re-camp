# Task Board Rule

## Status

- `Todo`: not started
- `Doing`: actively in progress
- `Review`: implemented and waiting for verification
- `Done`: verified and merged into the working branch
- `Blocked`: cannot proceed because of an external dependency

## Priority

- `P0`: blocks MVP loop
- `P1`: directly supports the MVP loop
- `P2`: important but can follow after MVP validation
- `P3`: polish, presentation, or expansion work

## Operating Rules

1. Keep one main task in `Doing` at a time.
2. Split work by gameplay slice, not by discipline alone.
3. Link each task to one milestone and one completion condition.
4. Mark a task `Done` only after local verification.
5. Move unclear work to `Blocked` with the missing dependency written down.

## Recommended Swimlanes

- Project Setup
- Game Flow
- Battle
- Reward Loop
- Camp
- UI/UX
- Art/Audio
- QA
