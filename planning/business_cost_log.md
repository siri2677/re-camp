# Re:Camp Project Cost Log

> 목적: Android 정식 출시까지 실제 현금 지출과 투입 시간을 기록해 프로젝트가 과도한 외주·구독·장비 비용으로 중단되는 것을 방지한다.

수익과 손익분기는 프로젝트 성공 조건이 아니다. 이 문서는 **완주 가능한 비용 범위를 유지하기 위한 관리표**다.

## 1. 현재 기준

```text
1차 제품 플랫폼: Google Play Android
개발·디버그: Windows PC
외부 테스트: APK 제한 배포·Google Play Closed Test
수익 우선순위: 낮음
무료 출시도 성공으로 인정
가챠·강제 광고·유료 재화 중심 설계 없음
```

배포 형태는 Android Vertical Slice 외부 테스트 후 결정한다.

```text
A. 무료 완성형
B. 저가 유료 앱
C. 무료 체험 + 1회 전체 해금
```

## 6. Gate별 비용 한도 원칙

### Android Touch Prototype 전

- 유료 Asset과 외주를 최소화한다.
- 기존 기기와 무료 도구로 핵심 루프를 먼저 검증한다.
- Store 등록 비용을 미리 집행하지 않는다.

### 루나 Character Proof 전

- 루나 제작에 직접 필요한 비용만 승인한다.
- 나머지 4명 최종 모델링 비용 금지
- 고가 Motion·VFX·UI Pack 선구매 금지

### Android Vertical Slice 전

- 최소한의 기준 Android 기기를 확보할 수 있다.
- 대규모 번역·마케팅·Store Art 비용 금지
- 무료 또는 기존 보유 음원·Asset으로 임시 검증 가능

### Google Play Closed Test 전

- 등록·서명·Store Listing에 필요한 비용 확인
- Low·Mid 기준 기기 호환성 부족 시 테스트 기기 보완
- 외부 QA는 내부 테스트로 해결할 수 없는 범위만 사용

### Android Full Release 전

- 남은 제작비와 출시 후 Patch·지원 비용 기록
- 혼자 감당할 수 없는 운영비가 발생하면 Scope를 축소
- 수익 예상보다 완성도와 유지 가능성을 우선

## 7. 유료 도구 승인 규칙

유료 도구를 추가할 때 기록한다.

```text
도구명
월·연 비용
필요한 작업
무료·기존 대체 수단
예상 절감 시간
종료 또는 해지 시점
생성 결과의 재사용 가능성
```

- 월 구독 도구를 필수 파이프라인으로 만들지 않는다.
- Aura AI는 기본 파이프라인에서 제외한다.
- 일회성 Proof를 위해 장기 구독을 시작하지 않는다.

## 8. 외주 승인 규칙

외주 전에 확인한다.

```text
Vertical Slice 또는 Full Release에 직접 필요한가
내부 제작 시 예상 시간
외주 비용
수정 횟수와 납품 형식
상업 사용 권리
Unity·Android 적용 가능 여부
다른 캐릭터·자산에 재사용 가능한가
```

루나 Proof 전에는 캐릭터 5명 전체 외주 계약을 하지 않는다.

## 10. 의사결정 규칙

- 비용이 증가하면 수익 모델을 공격적으로 바꾸기보다 Scope와 외주비를 먼저 검토한다.
- 광고·가챠·유료 재화를 비용 회수 수단으로 자동 추가하지 않는다.
- Android 제품 품질을 훼손하면서 PC·iOS 동시 출시비를 사용하지 않는다.
- 테스트 기기와 실제 APK 검증 비용은 외형 장식보다 우선할 수 있다.
- 계획 비용과 실제 비용을 분리한다.
- 근거 없이 높은 판매량을 가정하지 않는다.
