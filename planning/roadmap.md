# Re:Camp Integrated Roadmap

## 프로젝트 목표

Re:Camp는 **매력적인 캐릭터와 함께 폐허를 탐험하고, 자원을 회수해 캠프를 성장시키는 3D 쿼터뷰 로그라이트 RPG**다.

개발은 다음 두 축을 동시에 관리한다.

```text
게임 개발 축
→ 플레이 가능한 코어 루프, 전투, 성장, 저장, 최적화

캐릭터·아트 축
→ 5인 캐릭터 디자인, 2D 일러스트, 3D 모델, 환경, UI, VFX
```

최종 목표는 다음 범위의 소규모 정식 출시다.

| 항목 | 목표 |
|---|---:|
| 플랫폼 | PC 우선, 이후 Android 검토 |
| 플레이 방식 | 싱글플레이 |
| 캐릭터 | 루나·미유·코코·이리스·노아 5명 |
| 스테이지 | 2개 |
| 일반 몬스터 | 6종 |
| 보스 | 2종 |
| 캠프 시설 | 4종 이상 |
| 저장 | 로컬 저장 |
| 그래픽 | 3D 쿼터뷰, 툰 셰이딩 |
| 대표 일러스트 | 2D 일반 등신 |
| 인게임 캐릭터 | 3D 5~6등신 스타일라이즈드 |

## 핵심 우선순위

1. 전투 1판이 처음부터 끝까지 동작하는가
2. 캐릭터가 사용자에게 매력적으로 보이는가
3. 일러스트와 3D 모델이 같은 캐릭터로 보이는가
4. 캐릭터 5명의 전투 경험이 구분되는가
5. 작은 규모라도 완성된 게임으로 출시 가능한가

## M2. Unity 프로젝트·코어 통합 기반

### 목표

Unity 프로젝트를 생성하고 기존 Unity 비종속 게임 코어를 실제 프로젝트에 안전하게 연결한다.

### 작업

- Unity LTS 버전 확정과 프로젝트 생성
- Visible Meta Files, Force Text 설정
- `Assets/_ReCamp`, `Packages`, `ProjectSettings` 커밋
- 기본 Scene 5개 생성
- `SceneLoader`, Unity `GameManager` 구현
- 기존 `ReCamp.Core`의 Unity 호환성 검토
- Unity 호환 Assembly 또는 Adapter 구조 결정
- Core 상태를 Unity 화면과 입력에 연결
- EditMode·PlayMode 테스트 기반 추가
- CI에서 Core 테스트와 Unity 테스트 실행

### 완료 기준

```text
Unity 프로젝트가 Git에서 재현된다.
Lobby → Battle → Result → Lobby Scene 흐름이 동작한다.
기존 Core 규칙이 MonoBehaviour에 중복 구현되지 않는다.
```

## M3. 전투 Gray Box와 캐릭터 5명 플레이 프로토타입

### 목표

임시 Capsule과 Cube를 사용해 전투 코어와 캐릭터 5명의 플레이 차이를 검증한다.

### 공통 전투 작업

- PlayerController
- 카메라와 Input System
- 기본 공격과 Hitbox
- EnemyController와 EnemySpawner
- Damageable과 사망 처리
- 웨이브와 보스 등장
- 자원 드랍과 획득
- 귀환·사망·시간 종료
- 결과 정산

### 캐릭터 프로토타입

- 루나: 이동·회피·근접 연속 공격·자원 탐지
- 미유: 드론 자동 사격·대상 지정·자동 회수
- 코코: 회복·보호막·손실 감소
- 이리스: 차지 사격·약점·보스 추가 피해
- 노아: 정면 방어·타이밍 가드·반격

### 완료 기준

- 캐릭터 5명 모두 임시 모델로 선택·전투 가능
- 캐릭터별 조작감과 전투 역할이 겹치지 않음
- 한 판의 탐험·전투·정산 흐름이 완주 가능

아트 제작 전에 전투 역할을 먼저 검증해 디자인과 애니메이션 낭비를 줄인다.

---

## M4. 아트 파이프라인과 캐릭터 5명 2D 제작 시트

### 목표

모든 캐릭터의 최종 3D 모델링에 사용할 수 있는 Approved 2D 제작 시트를 확보한다.

### 파이프라인 작업

- ComfyUI 또는 동등한 이미지 워크플로 구성
- 캐릭터 기준 이미지·생성 변수·시드 관리
- Figma 캐릭터 시트 템플릿
- Blender MCP 테스트
- Coplay/Aura Import 테스트
- Git LFS와 `art_source/` 구성

### 캐릭터별 산출물

- 전신 Key Art
- 정면·측면·후면 Turnaround
- 얼굴 확대와 표정 8종
- 헤어·의상 분해도
- 무기·장비 상세도
- 컬러·재질표
- 대표 포즈
- 3D 제작 주의사항

### 완료 기준

루나·미유·코코·이리스·노아 모두 Approved 제작 시트를 보유하고, 5인을 함께 배치했을 때 같은 세계관과 서로 다른 매력이 동시에 보인다.

---

## M5. 루나 Character Proof와 Vertical Slice

### 목표

루나 1명과 스테이지 1개를 출시 수준으로 완성해 프로젝트 전체 품질과 제작 비용을 검증한다.

### 루나 제작

- 최종 2D 전신 일러스트
- 3D 5~6등신 모델
- 얼굴·헤어·의상·단검
- 공용 Humanoid Rig
- 표정 BlendShape
- Idle·Run·Attack·Dash·Skill·Hit·Victory
- 전용 툰 셰이더·VFX·사운드
- 로비·캐릭터 선택·전투 Prefab

### Vertical Slice 콘텐츠

- 버려진 거리 1개
- 일반 몬스터 3종
- 보스 1종
- 루나 스킬 6개 내외
- 캠프 시설 3종
- 로비·전투·결과·캠프 UI
- 저장·불러오기
- 튜토리얼
- 기본 사운드와 VFX

### 사용자 검증

- 캐릭터 첫인상과 기억도
- 일러스트와 3D 일치 여부
- 조작감과 타격감
- 5분 플레이 반복 의향
- 모바일 또는 목표 PC 성능

### 완료 기준

외부 사용자가 루나에게 매력을 느끼고, 한 판의 핵심 재미와 캠프 성장 동기를 이해할 수 있는 출시 품질의 작은 구간이 완성된다.

---

## M6. 공용 3D 기반과 나머지 캐릭터 제작

### 목표

루나에서 검증된 기반을 이용해 미유·코코·이리스·노아를 완성한다.

### 공용 기반

- 5~6등신 Base Mesh
- 공용 Humanoid Rig
- 공용 툰 셰이더
- 공용 Animator 구조
- 공용 이동·피격·사망 애니메이션
- 물리 본·Collider·LOD 규칙
- Unity 캐릭터 Prefab 템플릿

### 제작 순서

```text
미유 → 코코 → 이리스 → 노아
```

### 캐릭터별 고유 요소

- 얼굴·헤어·의상·무기
- 대표 Idle
- 공격·회피 또는 가드
- 스킬 3종
- 승리 모션
- 선택 연출
- VFX와 사운드 방향

### 완료 기준

5명 모두 선택·로비·전투에서 정상 동작하고, 각 캐릭터의 외형·모션·플레이 방식이 명확히 구분된다.

---

## M7. 전체 콘텐츠 완성

### 목표

출시 범위의 스테이지·적·캠프·캐릭터 콘텐츠를 모두 구현한다.

### 작업

- 두 번째 스테이지: 폐쇄된 연구소
- 일반 몬스터 총 6종
- 보스 총 2종
- 캠프 시설 4종 이상
- 캐릭터별 전용 스킬
- 캐릭터 해금 임무
- 친밀도와 개인 에피소드
- 캐릭터 대화와 로비 상호작용
- UI Design System과 모든 화면
- 환경·소품·VFX·사운드 완성

### 완료 기준

처음부터 마지막 캐릭터 해금과 두 번째 보스까지 중단 없이 플레이할 수 있다.

## M8. Release Candidate

### 목표

출시 가능한 안정성과 품질을 확보한다.

### 작업

- 전체 기능 회귀 테스트
- 저장 데이터 마이그레이션과 손상 방지
- 실제 기기·다양한 PC 사양 성능 테스트
- 메모리·GC·Draw Call·로딩 최적화
- 해상도·입력·키 설정 대응
- 캐릭터 관통·리깅·애니메이션 폴리시
- 아트 라이선스·AI 생성 메타데이터 검토
- 스토어 페이지·스크린샷·트레일러
- 체험판 또는 외부 베타 테스트

### 완료 기준

```text
치명적 진행 불가 버그가 없다.
저장 데이터가 안정적이다.
출시 대상 환경에서 목표 성능을 유지한다.
승인되지 않은 WIP 자산이 빌드에 포함되지 않는다.
캐릭터 5명이 사용자에게 서로 다른 매력으로 인식된다.
```
