# Re:Camp Milestones

## 확정된 제작 방향

- 장르: 모바일 로그라이트 생존 RPG
- 화면: SD 로우폴리 3D 캐릭터와 3D 배경
- 카메라: Orthographic 쿼터뷰
- 전투 좌표계: `XZ` 지면
- UI: Unity UGUI 기반 2D UI
- 렌더링: URP, 모바일 친화적 조명과 단순 셰이더
- 제작 원칙: 기능과 아트를 분리해서 끝내지 않고, 작은 버티컬 슬라이스 단위로 함께 완성한다.

## M0 - 프로젝트 기반 구축 (완료)

목표: Unity 프로젝트와 저장소 구조를 준비한다.

완료 조건:

- Unity 프로젝트와 기본 폴더 구조가 존재한다.
- 주요 씬과 공용 게임 흐름 구조가 존재한다.
- 기획·개발 문서 구조가 준비되어 있다.

## M1 - 코어 루프 프로토타입 (완료)

목표: `Lobby -> Battle -> Result -> Camp -> Lobby` 흐름을 검증한다.

완료 조건:

- 플레이어 이동과 자동 공격이 동작한다.
- 적 웨이브와 보스가 등장한다.
- 자원 드롭, 획득, 결과 정산이 동작한다.
- 캠프 강화와 로컬 저장이 동작한다.

## M2 - 3D 전투 기반 전환 (완료)

목표: 임시 `XY` 전투 구조를 최종 방향인 `XZ` 기반 3D 쿼터뷰 구조로 전환한다.

완료 조건:

- 플레이어·적·자원 이동 계산이 `XZ` 지면을 사용한다.
- Orthographic 쿼터뷰 카메라가 플레이어를 안정적으로 추적한다.
- 3D 바닥, 충돌, 캐릭터 회전 기준이 정리되어 있다.
- 기존 전투 루프가 전환 후에도 Play Mode에서 오류 없이 동작한다.
- 자동 회귀 테스트 또는 반복 가능한 MCP 점검 절차가 있다.

검증 기록:

- 카메라 기준 화면 상단 이동 벡터: `(0.707, 0, 0.707)`
- 화면 우측 이동 벡터: `(0.707, 0, -0.707)`
- 플레이어 경계 Clamp: `(8.50, 0.75, 8.00)`
- 전체 씬 루프와 보상 정산 후 `_ReCamp` Console 오류 `0건`

## M3 - 대표 전투 버티컬 슬라이스 (진행 중)

목표: 한 번의 전투가 임시 도형이 아닌 대표 비주얼로 플레이 가능하도록 만든다.

완료 조건:

- SD 로우폴리 플레이어 모델 1종이 적용되어 있다.
- 일반 적 2종과 보스 1종의 대표 모델이 적용되어 있다.
- 이동·공격·피격·사망 애니메이션의 최소 세트가 연결되어 있다.
- 대표 전투 배경 1종과 기본 조명이 적용되어 있다.
- 공격·피격·자원 드롭 VFX가 구분 가능하다.
- 전투 HUD가 UGUI로 교체되어 있다.

현재 HUD, 쿼터뷰 구성, 팔레트, 경계·환경 프록시와 대표 콘셉트는 완료됐다. 정식 모델·애니메이션·VFX·배경 에셋은 다음 제작 범위다.

## M4 - 캠프·보상 버티컬 슬라이스

목표: 귀환과 성장의 성취감을 대표 비주얼과 UI로 전달한다.

완료 조건:

- 캠프 대표 배경과 시설 3종이 시각적으로 구분된다.
- 결과 화면과 시설 강화 화면이 UGUI로 구현되어 있다.
- 보상 획득과 시설 강화 피드백이 제공된다.
- 저장·불러오기와 강화 효과가 회귀 검증된다.

## M5 - 데모 콘텐츠 완성

목표: 10~15분 분량의 대표 플레이 세션을 완성한다.

완료 조건:

- 3~5분 전투 한 판과 캠프 성장 루프가 자연스럽게 이어진다.
- 적 패턴, 보상량, 성장 비용이 데모 기준으로 조정되어 있다.
- 짧은 튜토리얼과 기본 조작 안내가 있다.
- BGM과 주요 SFX가 적용되어 있다.
- 모바일 해상도와 입력 환경에서 플레이 가능하다.

## M6 - 포트폴리오 폴리시 및 빌드

목표: 외부에 공개 가능한 안정적인 데모 빌드를 만든다.

완료 조건:

- 주요 화면 전환과 전투 연출이 일관된 품질을 가진다.
- 치명적인 오류와 진행 차단 문제가 없다.
- 성능, 로딩, 저장 데이터가 최종 점검되어 있다.
- Windows 또는 Android 데모 빌드가 생성된다.
- 스크린샷, 영상, 시스템 설명 자료가 준비되어 있다.
