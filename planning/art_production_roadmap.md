# Re:Camp Art Production Roadmap

## 목표

Re:Camp의 아트 제작 목표는 캐릭터 수를 줄이는 것이 아니라, **루나·미유·코코·이리스·노아 5명의 기획과 비주얼을 함께 확정하고 공용 제작 기반을 사용해 순차적으로 출시 품질까지 완성하는 것**이다.

```text
2D 일반 등신 일러스트
→ 캐릭터별 제작 시트
→ 3D 5~6등신 Blockout
→ 공용 Rig·Shader·Prefab 기반
→ 캐릭터별 최종 모델·애니메이션
→ Unity 쿼터뷰 검증
```

## A0. 아트 방향 통일

### 상태

Review

### 완료된 작업

- 캐릭터 5명의 이름과 역할 통일
- 2D 일반 등신 일러스트 방향 확정
- 3D 5~6등신 스타일라이즈드 인게임 방향 확정
- 툰 셰이딩·쿼터뷰·부드러운 포스트 아포칼립스 방향 확정
- 통합 캐릭터 라인업 1차 시안 생성
- 캐릭터 바이블과 MCP 제작 가이드 작성

### 남은 완료 기준

- 통합 라인업 시안을 사용자 검토 후 Approved 또는 수정 결정
- 캐릭터별 핵심 실루엣·색상·무기 설정 최종 승인

---

## A1. 아트 제작 파이프라인 구축

### 목표

캐릭터와 환경을 반복 제작할 수 있도록 MCP와 파일 관리 기준을 준비한다.

### 작업

- ComfyUI 또는 동등한 이미지 워크플로 구성
- 캐릭터별 기준 이미지와 생성 설정 저장 방식 정의
- Figma 캐릭터 시트 템플릿 구성
- Blender MCP 연결과 Export 규칙 검증
- Coplay/Aura의 Unity Import·Prefab 구성 검증
- Git LFS와 아트 소스 디렉터리 구성
- WIP / REVIEW / APPROVED 승인 흐름 적용

### 완료 기준

```text
루나 단독 이미지를 같은 얼굴·헤어·의상 설정으로 반복 생성할 수 있다.
Figma 캐릭터 시트 템플릿에 결과물을 정리할 수 있다.
Blender 테스트 에셋을 Unity로 Import하고 Prefab으로 구성할 수 있다.
```

---

## A2. 캐릭터 5명 동시 콘셉트 확정

### 목표

다섯 캐릭터 모두 모델링을 시작할 수 있는 수준의 2D 제작 시트를 확보한다.

### 캐릭터별 필수 산출물

- Key Art
- 정면·측면·후면 Turnaround
- 얼굴 확대
- 표정 8종
- 헤어 구조
- 의상 분해도
- 무기·장비 상세도
- 컬러·재질 팔레트
- 대표 포즈 3종
- 3D 제작 주의사항

### 진행 방식

```text
5명 디자인 탐색과 비교는 함께 진행
→ 캐릭터별 단독 시안 확정
→ 개별 제작 시트 순차 완성
→ 5명을 한 화면에 배치해 최종 일관성 검토
```

### 완료 기준

- 5명 모두 Approved 제작 시트 보유
- 얼굴·헤어·의상·무기 구조가 시점마다 일치
- 캐릭터별 색상과 실루엣이 겹치지 않음
- 전투 역할과 성격이 외형에서 구분됨

---

## A3. 루나 품질 기준 제작

### 목표

루나를 첫 번째 최종 캐릭터로 제작하여 나머지 4명의 품질과 제작 비용을 검증한다.

### 작업

- 최종 2D 전신 일러스트
- 3D 5~6등신 모델
- 얼굴·헤어·의상·에너지 단검
- 공용 Humanoid Rig 적용
- 얼굴 표정과 기본 BlendShape
- 대표 Idle, Run, Attack, Dash, Skill, Hit, Victory
- 툰 셰이더와 캐릭터 전용 VFX
- Unity 로비·전투 Prefab 통합
- 실제 쿼터뷰와 대상 기기 검증

### 완료 기준

- 일러스트와 3D 모델이 같은 캐릭터로 보임
- 쿼터뷰에서 얼굴·후드·단검이 식별됨
- 이동·공격·스킬에서 의상과 헤어 관통이 치명적이지 않음
- 사용자 테스트에서 캐릭터 매력에 대한 긍정 평가 확보

루나가 기준 품질을 충족하기 전에는 나머지 캐릭터의 최종 모델링을 대량 진행하지 않는다.

---

## A4. 공용 3D 제작 기반

### 목표

캐릭터 5명의 제작 비용과 유지보수 비용을 줄이는 공용 구조를 만든다.

### 작업

- 5~6등신 공용 신체 베이스
- 공용 Humanoid Rig
- 공용 얼굴·눈·헤어 툰 셰이더
- 공용 Material 슬롯 규칙
- 공용 Animator Controller 구조
- 공용 Idle·Walk·Run·Hit·Down 애니메이션
- 물리 본과 Collider 규칙
- 폴리곤·텍스처·LOD·머티리얼 규격
- Unity 캐릭터 Prefab 템플릿

### 완료 기준

루나 모델을 기준으로 공용 구조를 복제해 다른 캐릭터 Blockout을 만들 수 있고, Unity에서 동일한 이동·피격 애니메이션을 재사용할 수 있다.

---

## A5. 나머지 캐릭터 최종 제작

### 권장 제작 순서

```text
미유 → 코코 → 이리스 → 노아
```

### 캐릭터별 고유 제작 요소

- 얼굴과 머리카락
- 의상과 무기
- 대표 Idle
- 기본 공격
- 스킬 3종
- 승리 모션
- 캐릭터 선택 연출
- 전용 VFX
- 2D 전신 일러스트와 표정 일러스트

### 완료 기준

- 다섯 캐릭터 모두 선택·로비·전투에서 정상 동작
- 캐릭터 간 플레이 방식과 실루엣이 구분됨
- 공용 기반을 사용하지만 대표 모션과 공격에서 개성이 드러남

## A6. 환경·몬스터·소품

### 출시 목표

- 스테이지 2개
- 일반 몬스터 6종
- 보스 2종
- 캠프 시설 4종 이상
- 자원·상자·바리케이드·건물 모듈

### 제작 우선순위

```text
Gray Box 환경
→ 캠프 핵심 시설
→ 버려진 거리
→ 몬스터 3종과 보스 1종
→ 폐쇄된 연구소
→ 추가 몬스터 3종과 보스 1종
→ 환경 폴리시
```

배경은 캐릭터보다 낮은 채도와 시각적 밀도를 유지한다.

## A7. UI·VFX·연출

### UI

- 타이틀·로비
- 캐릭터 선택
- 캐릭터 정보·친밀도
- 전투 HUD
- 스킬 선택
- 결과·정산
- 캠프 강화
- 대화 UI

### VFX

- 루나: 민트·청록 베기
- 미유: 라일락·네온 블루 드론·홀로그램
- 코코: 코랄·그린 회복·보호막
- 이리스: 퍼플·크림슨 조준·정밀 사격
- 노아: 네이비·앰버 방벽·반격

### 완료 기준

모바일 화면에서 전투 정보가 읽히고, 이펙트만 보아도 어떤 캐릭터의 공격인지 구분된다.

## A8. 출시 아트 폴리시

### 작업

- 캐릭터 5명 최종 일관성 리뷰
- 얼굴·헤어·의상·무기 리터칭
- 애니메이션 타이밍과 타격감 개선
- 물리 본과 관통 수정
- LOD·Texture·Material 최적화
- 다양한 화면비 검증
- 스토어 이미지와 트레일러용 Key Art
- 라이선스와 AI 생성 메타데이터 검토

### 완료 기준

- 출시 대상 기기에서 목표 프레임 유지
- 승인되지 않은 WIP 자산이 게임 빌드에 포함되지 않음
- 모든 최종 자산의 출처·라이선스·버전 확인 가능
- 다섯 캐릭터가 사용자에게 서로 다른 매력과 역할로 인식됨
