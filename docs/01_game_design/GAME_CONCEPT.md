# Re:Camp Game Concept

## 1. 게임명

Re:Camp

## 2. 장르

- 가로형 모바일 3D 고정 쿼터뷰 로그라이트 RPG
- 생존 탐험과 귀환
- 캠프 거점 성장
- 서브컬쳐 캐릭터 수집·성장
- 싱글플레이·로컬 저장

## 3. 한 줄 소개

멸망 이후 자연에 잠식된 위험 구역을 3~5분 동안 탐험하고 자원을 회수해 생존자 캠프와 캐릭터를 성장시키는 모바일 로그라이트 RPG.

## 4. 기획 의도

```text
짧은 탐험의 긴장감
+ 더 탐험할지 귀환할지 선택하는 리스크·보상
+ 캠프가 성장하는 누적 성취감
+ 매력적인 캐릭터를 직접 조작하는 애착
```

플레이어는 위험 구역에서 몬스터를 피하거나 처치하고 자원을 수집한다. 오래 머물수록 보상과 위험이 함께 증가한다. 귀환하면 획득 자원을 캠프 시설에 투자해 다음 탐험을 준비한다.

## 5. 핵심 재미

1. 제한된 시간 안에 경로와 전투를 판단하는 탐험
2. 더 모을지 안전하게 귀환할지 결정하는 선택
3. 공격·회피·스킬 조합으로 달라지는 전투
4. 캠프 시설이 실제 편의와 전투력으로 이어지는 성장
5. 외형·성격·역할이 분명한 캐릭터에 대한 애착
6. 짧은 한 판이 다음 모바일 세션으로 이어지는 반복성

## 6. 플레이 시간 설계

```text
1회 탐험: 3~5분
캠프 정비: 1~3분
권장 모바일 세션: 탐험 1~3회, 약 5~15분
```

긴 플레이를 강제하지 않는다. 캐릭터 해금, 캠프 시설 성장, 빌드, 난이도와 관계 진행을 통해 여러 짧은 세션이 장기 성장으로 연결된다.

## 7. 플레이어 경험 목표

| 경험 | 설명 |
|---|---|
| 긴장감 | 탐험 시간이 지날수록 적과 위험이 증가한다. |
| 선택 | 귀환·추가 탐험·위험 보상 중 하나를 선택한다. |
| 숙련 | 이동·회피·스킬 선택과 적 패턴 이해가 향상된다. |
| 성장 | 캐릭터와 캠프가 반복 플레이를 통해 강화된다. |
| 애착 | 캐릭터의 외형·모션·대사·역할을 기억한다. |
| 접근성 | 짧게 켜서 한 판을 끝내고 자연스럽게 종료할 수 있다. |

## 8. 플랫폼 경험

### Google Play Android

- 1차 제품·정식 출시 플랫폼
- 가로형 양손 Touch 조작
- 좌측 가상 조이스틱
- 우측 공격·대시·스킬 버튼
- Auto Aim 또는 대상 보정
- Safe Area와 화면비 대응
- 30 FPS 기본·지원 기기 60 FPS 선택

### Windows

- Unity 개발·디버깅·자동 테스트·Profiler 기준
- Keyboard/Mouse·Gamepad Adapter 제공
- 포트폴리오 영상·Screenshot과 필요 시 itch.io Demo
- 1차 제품 플랫폼이 아니며 PC판을 위해 모바일 핵심 구조를 변경하지 않음

### Steam·iOS

- Android Full Release와 유지보수 안정화 후 별도 제품화 비용을 평가한다.

## 9. 비주얼 톤

```text
2D 일반 등신 캐릭터 일러스트
+ 3D 5~6등신 스타일라이즈드 인게임 모델
+ 툰 셰이딩
+ 자연에 잠식된 파스텔 폐허
+ 따뜻한 캠프와 위협적인 탐험 지역의 대비
```

캐릭터의 매력과 작은 모바일 화면에서의 식별성을 배경의 사실성보다 우선한다.

## 10. 캐릭터 방향

플레이어블 캐릭터는 루나·미유·코코·이리스·노아 5명이다. 현재 통합 라인업은 역할·색상·실루엣의 `Direction Approved` 상태이며 개별 최종 제작 시트는 제작 전이다.

루나 Character Proof를 먼저 진행한 뒤 검증된 Base Mesh·Rig·Shader·Prefab을 나머지 캐릭터에 확장한다.

## 11. 차별화 포인트

```text
3~5분 직접 조작 로그라이트 전투
+ 탐험 후 귀환하는 명확한 리스크·보상
+ 캠프 성장으로 이어지는 장기 목표
+ 2D 서브컬쳐 디자인을 3D 모바일 플레이로 연결
+ 폐허와 희망적인 캠프의 정서적 대비
```

## 12. 1차 타깃

- 짧은 시간에도 한 판의 완결감을 원하는 모바일 RPG 유저
- 서브컬쳐 캐릭터와 직접 조작 전투를 선호하는 유저
- 생존·탐험·귀환 구조를 좋아하는 유저
- 복잡한 MMORPG보다 반복 가능한 소규모 RPG를 원하는 유저
- 출퇴근·휴식 시간·취침 전에 짧게 플레이하는 유저

## 13. 배포·수익 기준

```text
1차 플랫폼: Google Play Android
배포 목표: 정식 공개 가능한 완성형 게임
수익 우선순위: 낮음
가챠·강제 광고·유료 재화: 사용하지 않음
```

무료, 저가 유료, 무료 체험 후 1회 해금 중 어떤 형태를 사용할지는 Vertical Slice 외부 테스트 후 결정한다. 판매량과 수익은 출시 여부의 필수 조건이 아니다.

- 복잡한 장비 파밍과 의상 교체
- 대량 컷신과 풀 보이스
- Android와 PC 동시 정식 출시
