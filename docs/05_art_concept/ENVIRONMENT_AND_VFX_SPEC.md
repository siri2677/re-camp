# Re:Camp Environment and VFX Spec

> 상태: Review  
> 목적: Vertical Slice의 버려진 거리, 캠프, 몬스터, 자원, 전투 피드백을 같은 시각 언어로 제작한다.

## 1. 환경 아트 방향

```text
자연에 잠식된 폐허
+ 부드러운 포스트 아포칼립스
+ 캐릭터보다 낮은 채도와 시각적 밀도
+ 이동 경로와 상호작용 오브젝트의 명확한 가독성
```

환경은 사실적 파괴 묘사보다 캐릭터와 플레이 정보를 돋보이게 하는 무대로 사용한다.

## 2. Vertical Slice 환경 범위

### 버려진 거리

필수 모듈:

- 도로 직선·교차로·막힌 도로
- 인도·턱·배수구
- 건물 외벽·입구·파손 벽
- 차량 2종 이상
- 바리케이드·상자·잔해
- 덩굴·풀·관목·작은 나무
- 자원 스폰 지점
- 귀환 지점
- 전투 구역과 보스 구역

### 캠프

필수 시설:

- 발전기
- 작업대
- 식량 창고

장기 확장 시설:

- 통신탑
- 연구 시설
- 캐릭터 휴식 공간

## 3. 모듈 제작 기준

### Grid와 Pivot

- 기본 Grid 단위는 1m를 사용한다.
- 대형 환경 모듈은 2m 또는 4m 배수로 맞춘다.
- 바닥 모듈 Pivot은 하단 중앙 또는 결합 기준점에 둔다.
- 벽과 건물 모듈 Pivot은 반복 배치와 회전이 쉬운 모서리 기준을 우선한다.
- Prefab Scale은 `(1, 1, 1)`을 유지한다.

### Collision

- 시각 Mesh와 Collision Mesh를 분리할 수 있다.
- 이동 경로의 작은 잔해에는 불필요한 세부 Collider를 두지 않는다.
- 플레이어가 걸리는 틈과 보이지 않는 돌출부를 피한다.
- NavMesh 통로는 캐릭터와 적의 회피 폭을 고려해 확보한다.

### Material

- 반복되는 건물·도로·소품은 공유 Material과 Atlas를 우선한다.
- Decal로 균열·오염·표식을 변형해 반복감을 줄인다.
- 투명 Material은 식생과 제한적 효과에만 사용한다.
- 캐릭터보다 높은 채도와 강한 Emission을 환경에 상시 사용하지 않는다.

### LOD와 Culling

- 대형 건물, 차량, 식생 군집은 LOD 또는 단순 Proxy를 검토한다.
- 카메라가 고정되어 보이지 않는 내부 면은 제거한다.
- 작은 소품은 거리별 Culling을 적용한다.
- Occlusion Culling은 실제 Scene 구성 후 효과를 측정해 적용한다.

## 4. Color Key

### 버려진 거리

- 기본색: 회색·저채도 청록·낡은 베이지
- 자연: 부드러운 녹색과 황록색
- 위험: 제한된 적색·주황색
- 자원: 민트·청록·크리스털 발광
- 루나와 충돌하는 과도한 민트 배경 사용 금지

### 캠프

- 크림·따뜻한 주황 조명·목재색
- 시설의 작동 상태는 밝은 민트 또는 앰버로 표현
- 위험 구역과 명확히 대비되는 안전하고 따뜻한 분위기

## 5. 몬스터 실루엣

Vertical Slice 일반 몬스터 3종은 최소 다음 역할을 구분한다.

| 역할 | 시각적 방향 | 전투 읽기 |
|---|---|---|
| 근접 추적형 | 낮고 전방으로 기운 실루엣 | 돌진과 물기·베기 준비가 보임 |
| 원거리 공격형 | 상체 또는 발사 기관 강조 | 조준선과 투사체 준비가 보임 |
| 중장갑형 | 넓고 무거운 실루엣 | 약점과 방어 방향이 보임 |

보스는 일반 적의 단순 확대가 아니라 약점·공격 기관·위험 방향이 외형에서 읽혀야 한다.

## 6. 자원과 상호작용 오브젝트

- 고철: 금속 덩어리와 기계 부품, 청록 또는 흰색 윤곽
- 식량: 용기·보급품 형태, 따뜻한 색상
- 데이터 조각: 결정·메모리 장치, 보라·청록 발광
- 상자는 열림·미개봉 상태가 멀리서도 구분되어야 함
- 드롭 아이템은 Ground Beam, Icon, Outline 중 2개 이하를 조합해 과도한 발광을 피함

## 7. VFX 언어

### 공통 규칙

- 공격, 위험, 회복, 보호막, 자원 획득을 색상과 형태로 구분한다.
- 캐릭터와 적의 실루엣을 완전히 가리는 불투명 효과를 피한다.
- Hit Frame과 VFX 발생 시점을 Animation·Gameplay 데이터와 일치시킨다.
- 일반 공격은 짧고 선명하게, 강한 스킬은 더 넓고 오래 표현한다.
- VFX는 Pooling을 기본으로 한다.

### 캐릭터 색상

| 캐릭터 | VFX 색상·형태 |
|---|---|
| 루나 | 민트·청록 베기, 얇고 빠른 궤적 |
| 미유 | 라일락·네온 블루, 드론·홀로그램·기하학 |
| 코코 | 코랄·그린, 둥근 회복 파동·보호막 |
| 이리스 | 퍼플·크림슨, 조준선·집중된 직선 |
| 노아 | 네이비·앰버, 방벽·육각형·반격 충격파 |

### 시스템 색상

- 적 위험 장판: 빨강·주황
- 플레이어 회복: 초록 또는 청록
- 보호막: 청색·백색
- 자원: 종류별 고정 색상
- 상호작용 가능: 흰색·민트

캐릭터 VFX가 적 위험 장판과 혼동되지 않도록 색상뿐 아니라 형태·애니메이션 방향도 구분한다.

## 8. 루나 Vertical Slice VFX

```text
기본 공격: 0.2~0.35초 민트 단검 궤적
3타 마무리: 민트 중심 + 제한적 핑크 Accent
Dash: 짧은 잔상과 바닥 방향선
Scan Pulse: 청록 원형 파동과 탐지 대상 Highlight
Cross Fang: 교차하는 두 개의 명확한 궤적
Hit: 짧은 Hit Flash와 방향성 Spark
자원 획득: 작고 빠른 흡수 궤적
```

## 9. 피드백 연출

- Hit Stop은 강한 타격에만 제한적으로 사용한다.
- Camera Shake는 강도 단계와 접근성 설정을 둔다.
- Damage Number는 화면을 과도하게 채우지 않도록 합산 또는 제한한다.
- 적 공격은 Telegraph → Active → Recovery가 구분되어야 한다.
- 보스 위험 공격은 색상 외에 형태·범위·사운드로도 전달한다.

## 10. 성능 검수

측정 항목:

```text
동시 Particle 수
투명 Overdraw
VFX Draw Call
GPU Frame Time
동시 활성 Light 수
Trail·Line Renderer 수
Pooling Hit Rate
```

High / Medium / Low 품질에 따라 Particle 수, Trail 길이, Distortion, Light, Decal 수를 조정한다.

## 12. 승인 조건

- 버려진 거리 Color Key가 캐릭터 가독성을 유지함
- Modular Kit으로 Gray Box를 대체할 수 있음
- 몬스터 3종과 보스의 역할이 실루엣에서 구분됨
- 루나의 민트 VFX와 적 위험 표시가 혼동되지 않음
- Windows 목표 성능을 충족함
- Low 품질에서도 공격·위험·자원이 명확히 읽힘
